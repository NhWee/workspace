compare_wave_datasets.py 가이드
===============================

이 파일은 여러 `.npz` wave dataset의 metadata와 기본 수치 범위를 Markdown 표로 비교하는 CLI입니다.

시뮬레이션 크기, frame 개수, velocity 저장 여부, Git commit, 수면/수심/속도 범위를 한눈에 볼 수
있어서 여러 run을 비교하거나 재현성을 확인할 때 유용합니다.


1. 무엇을 만든 것인가
---------------------

compare_wave_datasets.py는 다음 정보를 읽습니다.

- dataset path
- size
- steps
- frame_every
- frame_count
- stores_velocity
- device
- created_at_utc
- git_commit
- git_is_dirty
- eta_min / eta_max
- depth_min / depth_max
- speed_max
- final_l2_vs_baseline
- final_linf_vs_baseline
- final_diff_heatmap
- frames_l2_mean_vs_baseline
- frames_linf_max_vs_baseline
- frame_metrics_csv

`final_l2_vs_baseline`과 `final_linf_vs_baseline`은 첫 번째 dataset의 마지막 수면 frame을
baseline으로 삼아 계산합니다. grid shape이 다르면 `-`로 표시합니다.

`frames_l2_mean_vs_baseline`과 `frames_linf_max_vs_baseline`은 전체 frame tensor의 shape이
같을 때만 계산합니다.


2. 실행 방법
------------

하나의 dataset 요약:

    .\.venv\Scripts\python.exe compare_wave_datasets.py outputs\wave_dataset_velocity.npz

여러 dataset 비교:

    .\.venv\Scripts\python.exe compare_wave_datasets.py outputs\wave_dataset.npz outputs\wave_dataset_velocity.npz

출력 경로 지정:

    .\.venv\Scripts\python.exe compare_wave_datasets.py outputs\wave_dataset_velocity.npz --output outputs\my_comparison.md

final frame 차이 heatmap 저장:

    .\.venv\Scripts\python.exe compare_wave_datasets.py outputs\run_a.npz outputs\run_b.npz --diff-heatmap-dir outputs\diff_heatmaps

frame별 L2/Linf CSV 저장:

    .\.venv\Scripts\python.exe compare_wave_datasets.py outputs\run_a.npz outputs\run_b.npz --frame-metrics-dir outputs\frame_metrics

frame별 L2/Linf Plotly chart 저장:

    .\.venv\Scripts\python.exe compare_wave_datasets.py outputs\run_a.npz outputs\run_b.npz --frame-metrics-chart outputs\frame_metrics.html


3. 출력 파일
------------

기본 출력:

    outputs\wave_dataset_comparison.md

같은 Markdown table이 터미널에도 출력됩니다.

`--diff-heatmap-dir`를 지정하면 baseline과 같은 grid shape인 dataset에 대해
`abs(final_eta - baseline_final_eta)` PNG도 저장합니다.

`--frame-metrics-dir`를 지정하면 baseline과 전체 frame tensor shape이 같은 dataset에 대해
frame별 L2/Linf CSV를 저장합니다.

`--frame-metrics-chart`를 지정하면 같은 frame별 L2/Linf series를 Plotly HTML chart로 저장합니다.


4. 왜 중요한가
--------------

3D wave workflow에서는 같은 solver로 여러 size, steps, seed, viewer 설정을 실험하게 됩니다.

이 도구는 다음을 빠르게 확인하게 해줍니다.

- 어떤 dataset이 velocity field를 포함하는지
- 어느 commit에서 dataset이 만들어졌는지
- run 사이 수면 높이, 수심, 최대 속도가 크게 달라졌는지
- 비교할 viewer 입력을 잘 골랐는지


5. 현재 검증
------------

다음 검증을 완료했습니다.

- metadata provenance field 읽기
- frame/depth 기본 수치 범위 계산
- velocity dataset의 speed_max 계산
- 같은 grid shape dataset의 final frame L2/Linf 차이 계산
- 같은 frame tensor shape dataset의 frame-wise L2/Linf summary 계산
- 같은 frame tensor shape dataset의 frame-wise L2/Linf CSV 저장
- 같은 frame tensor shape dataset의 frame-wise L2/Linf Plotly chart 저장
- 같은 grid shape dataset의 final frame difference heatmap 저장
- Markdown comparison table 생성
- validate_wave_workflow.py에서 comparison markdown 생성 확인
- py_compile 통과
- pip check 통과


6. 현재 한계
------------

- 비교 결과는 요약 metric, frame별 CSV, Plotly chart 중심이며 animation은 아직 만들지 않습니다.
- dataset shape이 서로 다르면 차이 metric은 계산하지 않습니다.
- 큰 dataset에서도 전체 배열을 읽어 통계를 계산합니다.

8. 한 줄 요약
--------------

compare_wave_datasets.py는 여러 wave dataset의 provenance와 기본 물리 수치를 표로 비교해,
GPU 파도 실험을 재현하고 비교하는 기반을 제공합니다.
