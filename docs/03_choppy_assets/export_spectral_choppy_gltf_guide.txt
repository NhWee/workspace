export_spectral_choppy_gltf.py 가이드
====================================

이 스크립트는 GPU FFT 기반 choppy wave simulation의 마지막 frame을 glTF 2.0
scene으로 저장합니다.


1. 무엇을 만드는가
------------------

기본 출력은 embedded `.gltf` 파일 하나입니다. `--glb-output`를 지정하면 같은
final frame을 binary `.glb`로도 저장합니다. `--sequence-output-dir`나
`--glb-sequence-output-dir`를 지정하면 저장된 모든 frame을 개별 scene 파일로
내보내고 manifest를 함께 저장합니다.
`--animated-output`나 `--animated-glb-output`를 지정하면 저장된 frame을
morph target animation으로 묶은 단일 animated scene도 저장합니다.

- water surface: triangle mesh primitive
- water normal: vertex normal attribute
- foam layer: point primitive
- binary data: base64 data URI로 glTF 내부에 포함
- GLB: JSON chunk와 binary chunk를 하나의 binary 파일로 저장
- sequence manifest: frame별 glTF 파일 경로와 요약 정보
- animated glTF/GLB: 저장된 water surface와 foam point frame을 morph target weight animation으로 재생

OBJ/PLY export는 파일을 역할별로 나누는 방식이고, glTF export는 final frame의
표면과 foam point를 하나의 scene 파일로 묶는 방식입니다.


2. 사용한 geometry
------------------

water surface:

- `POSITION`: choppy displacement가 적용된 `(x, y, z)`
- `NORMAL`: 중심 차분 tangent의 cross product로 계산한 vertex normal
- `indices`: quad grid를 두 개의 triangle로 분할한 index buffer
- primitive mode: `TRIANGLES`

foam layer:

- `POSITION`: steepness 기반 foam point 위치
- `_STEEPNESS`: custom scalar attribute
- primitive mode: `POINTS`


3. 실행 방법
------------

기본 실행:

    .\.venv\Scripts\python.exe export_spectral_choppy_gltf.py

작은 smoke test:

    .\.venv\Scripts\python.exe export_spectral_choppy_gltf.py --size 64 --steps 48 --frame-every 12 --max-surface-points 32 --foam-threshold 0.0 --max-foam-points 64 --output outputs\spectral_choppy_gltf_smoke.gltf

고해상도 final frame export:

    .\.venv\Scripts\python.exe export_spectral_choppy_gltf.py --size 512 --steps 720 --frame-every 24 --max-surface-points 256 --output outputs\spectral_choppy_wave_final.gltf --glb-output outputs\spectral_choppy_wave_final.glb

glTF sequence export:

    .\.venv\Scripts\python.exe export_spectral_choppy_gltf.py --size 128 --steps 120 --frame-every 12 --max-surface-points 64 --sequence-output-dir outputs\spectral_choppy_gltf_sequence

GLB sequence export:

    .\.venv\Scripts\python.exe export_spectral_choppy_gltf.py --size 128 --steps 120 --frame-every 12 --max-surface-points 64 --glb-sequence-output-dir outputs\spectral_choppy_glb_sequence

animated GLB export:

    .\.venv\Scripts\python.exe export_spectral_choppy_gltf.py --size 128 --steps 120 --frame-every 12 --max-surface-points 64 --animated-output outputs\spectral_choppy_wave_animated.gltf --animated-glb-output outputs\spectral_choppy_wave_animated.glb


4. 주요 옵션
------------

- `--size`: 내부 FFT simulation grid 크기
- `--steps`: simulation step 수
- `--frame-every`: frame 저장 간격
- `--max-surface-points`: glTF로 내보낼 한 축 최대 surface point 수
- `--choppiness`: 수평 displacement 강도
- `--foam-threshold`: foam point로 내보낼 최소 steepness
- `--max-foam-points`: 최대 foam point 수
- `--foam-z-offset`: foam point를 수면 위로 올리는 z offset
- `--output`: glTF 저장 경로
- `--glb-output`: final frame GLB 저장 경로
- `--sequence-output-dir`: 저장된 모든 frame을 glTF sequence로 내보낼 폴더
- `--glb-sequence-output-dir`: 저장된 모든 frame을 GLB sequence로 내보낼 폴더
- `--animated-output`: morph target animated glTF 저장 경로
- `--animated-glb-output`: morph target animated GLB 저장 경로
- `--animation-frame-duration`: animated scene의 저장 frame 간격. 기본값은 `frame_every * dt`


5. 한계
-------

final glTF/GLB는 final frame scene입니다. animated glTF/GLB는 water surface와 foam point morph target animation입니다.

- foam은 point primitive이며 실제 유체 입자 simulation은 아닙니다.
- animated foam은 고정 point capacity에 맞춘 steepness 기반 point primitive입니다.
- embedded base64 방식이므로 큰 grid에서는 파일 크기가 빠르게 커집니다.
- sequence export는 animation clip이 아니라 frame별 scene 파일 묶음입니다.
