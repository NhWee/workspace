What this makes
---------------
This script is a first 2D Navier-Stokes experiment for vortex-driven foam.
It does not yet simulate a 3D free water surface. Instead, it simulates a
top-down 2D incompressible velocity field and shows where foam-like material
would be generated by strong swirling motion.

The output HTML has two panels:

1. vorticity / swirl
   This shows curl(u, v). Bright positive and negative regions are rotating
   fluid structures.

2. foam scalar
   This shows a visual foam field. Foam is generated where speed and vorticity
   are high, then it is advected by the fluid and fades over time.

Equations
---------
The velocity field follows the 2D incompressible Navier-Stokes equations:

du/dt + (u dot grad)u = -grad(p) + viscosity * laplacian(u) + force
div(u) = 0

Here:

u, v       = horizontal velocity components
p          = pressure
viscosity  = how quickly small swirls are smoothed out
force      = artificial rotating sources that keep vortices moving

The code uses a common "stable fluids" style loop:

1. advect velocity through itself
2. add rotating vortex forces
3. diffuse velocity with viscosity
4. solve pressure with Jacobi iterations
5. subtract pressure gradient so div(u) is close to zero
6. advect and update foam

Foam model
----------
Foam is not a full physical bubble simulation yet. It is a scalar visualization:

foam_source = high_vorticity * high_speed
foam_next = advected_foam * decay + foam_birth * foam_source * dt

This is useful because breaking waves and white water often appear near strong
velocity gradients, vortices, and energetic flow regions.

Useful parameters
-----------------
--size
Grid resolution. Higher values look smoother but are heavier.

--steps
Total simulation steps.

--frame-every
Saves one animation frame every N simulation steps. Lower values are smoother
but create larger HTML files.

--dt
Simulation time step. Smaller is more stable and smoother. Larger is more
energetic but can become unstable.

--viscosity
How much the fluid smooths itself. Lower values preserve sharper vortices.
Higher values look thicker and calmer.

--pressure-iters
How many Jacobi iterations are used for pressure projection. Higher values make
the velocity field more incompressible but cost more time.

--force-strength
Strength of the rotating force sources. Higher values create more violent
swirls and more foam.

--force-radius
Size of the rotating force sources.

--foam-vorticity-threshold
How much curl is needed before foam is generated. Lower values create foam more
easily.

--foam-speed-threshold
How much speed is needed before foam is generated. Lower values create foam
more easily.

--foam-birth
Foam generation rate.

--foam-decay
Foam fade-out rate. Lower values make foam persist longer.

Example
-------
From the project root:

python -m wave_sim.navier_stokes.navier_stokes_foam_2d

Heavier/smoother example:

python -m wave_sim.navier_stokes.navier_stokes_foam_2d --size 256 --steps 1200 --frame-every 3 --pressure-iters 80 --force-strength 5.2 --foam-decay 0.42
