What this makes
---------------
This is the first step from the 2D Navier-Stokes foam experiment toward a 3D
water surface.

It still solves the fluid motion on a 2D horizontal grid, but it adds a height
field:

eta(x, y, t)

The result is rendered as a 3D surface. Foam is drawn into the surface color, so
bright regions indicate energetic white-water-like areas.

Equations
---------
The horizontal velocity field uses the 2D incompressible Navier-Stokes form:

du/dt + (u dot grad)u = -grad(p) + viscosity * laplacian(u) + force
div(u) = 0

The free surface is a height field coupled to the flow:

d2 eta/dt2 =
    wave_speed^2 * laplacian(eta)
    + surface_coupling * vortex_source
    - surface_damping * d eta/dt

eta is also advected by the horizontal velocity field, so the surface pattern is
carried around by the flow.

Foam model
----------
Foam is generated from:

1. high vorticity
2. high speed
3. elevated crests

This is not yet a full bubble/air-water simulation. It is a useful visual and
algorithmic step toward white water driven by the fluid state.

Important parameters
--------------------
--scene
Reusable scene preset. Available values:

none
No scene preset. Use raw parameter defaults and command-line values.

smooth_long
Lightweight, longer, smoother playback.

dynamic_splash
More energetic initial flow and stronger splash particles. Analysis overlays
are off by default for a more realistic view.

vortex_focus
Emphasizes visible vortex spirals and high-vorticity regions.

rock_wake
Incoming waves hit a circular rock obstacle and create wake, streaks, foam, and
splash. This is the recommended scene for a more turbulent realistic view.

Scene presets can still be overridden. For example, --scene dynamic_splash
--fps 30 keeps the dynamic_splash setup but changes playback FPS.

The smooth_long, dynamic_splash, and vortex_focus scene presets enable periodic
forcing by default, so they keep receiving wave energy over time.

--size
Simulation resolution.

--steps
Total simulation length.

--frame-every
Lower values save more animation frames and look smoother.

--dt
Simulation time step.

--viscosity
Higher values smooth the flow. Lower values keep sharper vortices.

--pressure-iters
Higher values make the velocity field more incompressible, but cost more time.

--force-strength
Strength of the rotating flow sources.

--force-radius
Radius of the rotating flow sources.

--periodic-force / --no-periodic-force
Turns repeating external wave forcing on or off. This keeps injecting wave
energy from one side so the state does not rely only on the initial condition.

--periodic-force-strength
Horizontal strength of the repeating external wave forcing.

--periodic-surface-strength
Vertical strength applied to the free-surface motion.

--periodic-wavelength
Wavelength of the incoming periodic forcing pattern.

--periodic-period
Temporal period of the incoming wave forcing. Lower values make incoming waves
arrive faster.

--periodic-direction-degrees
Direction of the incoming wave forcing.

--periodic-inlet-width
Width of the side region where the periodic wave enters the domain.

--obstacle / --no-obstacle
Adds a circular rock obstacle. The flow is damped inside the obstacle, and the
wake region drives stronger foam and surface streaks.

--obstacle-x / --obstacle-y / --obstacle-radius
Position and size of the circular obstacle.

--obstacle-wake-strength
How strongly the obstacle wake contributes to foam and streaks.

--vorticity-confinement
Adds energy back into vortices so turbulent structures remain visible longer.

--streak-strength
Strength of the advected surface streak texture. This makes the flow direction
more visible without adding analysis marker lines.

--streak-decay
How quickly the advected streak texture fades.

--wave-speed
How quickly surface height ripples propagate.

--surface-coupling
How strongly vortices push/pull the surface.

--surface-damping
How quickly vertical surface motion settles down.

--surface-smoothing
Per-step smoothing applied to eta. Higher values reduce fast high-frequency
fluctuation, but too much smoothing makes the surface look thick and slow.

--max-eta-velocity
Clamp for vertical surface velocity. This helps prevent sudden sharp flicker.
Use 0 to disable the clamp.

--eta-scale
Visual vertical scale of the rendered surface.

--foam-vorticity-threshold
How much swirl is needed before foam appears.

--foam-speed-threshold
How much speed is needed before foam appears.

--foam-birth
How quickly foam is created.

--foam-decay
How quickly foam fades.

--foam-particles / --no-foam-particles
Turns the 3D foam particle layer on or off. The surface color still shows foam.

--particle-life
Average lifetime of foam particles.

--particle-spawn-per-frame
Approximate number of foam particles created per rendered frame.

--max-particles
Maximum active foam particles.

--particle-height
Small vertical offset that places foam particles above the water surface.

--particle-size
Rendered marker size scale for foam particles.

--splash-particles / --no-splash-particles
Turns upward splash particles on or off. Splash particles are emitted from
energetic foam/vorticity regions and then fall under gravity.

--splash-spawn-per-frame
Approximate number of splash particles created per rendered frame.

--max-splash-particles
Maximum active splash particles.

--splash-life
Average splash particle lifetime.

--splash-gravity
Downward acceleration applied to splash particles.

--splash-threshold
Minimum energetic source value before splash appears.

--splash-burst-min / --splash-burst-max
Minimum and maximum upward velocity for newly emitted splash particles.

--splash-spread
Horizontal random spread for splash particles.

--splash-size
Rendered splash particle size scale.

--vortex-markers / --no-vortex-markers
Turns high-vorticity analysis marker points on or off. Keep this off for
realistic output.

--max-vortex-markers
Maximum vortex markers per frame.

--vortex-size
Rendered vortex marker size scale.

--vortex-spirals / --no-vortex-spirals
Turns visible analysis spiral line overlays on or off. Spiral lines make the
main vortices easier to inspect, but they are not part of the physical water
surface. Keep this off for realistic output.

--vortex-spiral-count
Number of visible vortex spirals.

--vortex-spiral-points
Points per rendered spiral.

--vortex-spiral-radius
Radius of each rendered vortex spiral.

--vortex-line-width
Rendered line width for vortex spirals.

--quality
Output preset. preview is faster and lighter, balanced is the default, and
final keeps more frames/detail.

--fps
Viewer playback FPS. If this is provided, it overrides --frame-duration-ms.

--frame-duration-ms
Viewer frame duration in milliseconds. Lower values play faster. Higher values
play slower and make fast fluctuation easier to inspect.

Example
-------
From the project root:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d

Use a reusable scene preset:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --scene dynamic_splash

Turbulent incoming waves around a rock obstacle:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --scene rock_wake

Smoother output:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --frame-every 2 --frame-duration-ms 25

Slower playback for checking fast fluctuation:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --fps 24

HTML controls
-------------
Replay starts again from the first frame. Use this after the animation has
already reached the end.

Play continues from the current slider position.

Pause stops the animation.

The frame slider can be dragged to inspect a specific moment before pressing
Play or Replay.

More energetic surface:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --force-strength 5.2 --surface-coupling 0.75 --wave-speed 0.22

Calmer surface with clearer foam particles:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --fps 24 --wave-speed 0.10 --surface-coupling 0.35 --surface-damping 1.2 --surface-smoothing 0.12 --particle-spawn-per-frame 65

More splash and visible vortex markers:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --fps 24 --size 64 --steps 720 --frame-every 2 --max-surface-points 64 --particle-spawn-per-frame 25 --splash-spawn-per-frame 18 --max-splash-particles 500 --max-vortex-markers 80

Use this only as an analysis view:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --scene dynamic_splash --vortex-markers --vortex-spirals

More dynamic initial flow, stronger splash, and visible vortex spirals:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --fps 24 --size 64 --steps 720 --frame-every 2 --pressure-iters 18 --max-surface-points 64 --force-strength 5.8 --force-radius 0.15 --wave-speed 0.12 --surface-coupling 0.48 --surface-damping 1.05 --surface-smoothing 0.10 --particle-spawn-per-frame 28 --splash-spawn-per-frame 28 --max-splash-particles 650 --splash-burst-max 1.8 --splash-spread 0.085 --vortex-spiral-count 5 --vortex-spiral-radius 0.13 --vortex-line-width 7

The same direction using a preset:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --scene dynamic_splash

Incoming periodic waves with a calmer scene:

python -m wave_sim.navier_stokes.navier_stokes_free_surface_3d --scene smooth_long --periodic-force --periodic-surface-strength 0.035 --periodic-period 1.7

How this connects to the final goal
-----------------------------------
This is not yet a full 3D Navier-Stokes free-surface solver like VOF, FLIP, or
SPH. It is a practical intermediate stage:

1. keep a fast 2D Navier-Stokes velocity field
2. add a 3D height-field free surface
3. drive foam from vorticity, speed, and crests

The next step would be adding obstacles, shoreline/bathymetry, or particle foam
on top of this 3D surface.
