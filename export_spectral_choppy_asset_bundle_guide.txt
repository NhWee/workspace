export_spectral_choppy_asset_bundle.py 가이드
============================================

이 스크립트는 GPU FFT 기반 choppy wave simulation을 한 번 실행한 뒤, 3D workflow에
필요한 주요 산출물을 한 디렉터리에 묶어 저장합니다.


1. 무엇을 만드는가
------------------

기본 출력 폴더는 `outputs/spectral_choppy_asset_bundle`입니다.

- `viewer.html`: Plotly interactive viewer
- `final.obj`: final frame OBJ mesh
- `obj_sequence/`: frame별 OBJ sequence
- `foam.ply`: final frame foam point cloud
- `foam_sequence/`: frame별 foam PLY sequence
- `final.gltf`: final frame embedded glTF scene
- `final.glb`: final frame binary GLB scene
- `gltf_sequence/`: frame별 glTF sequence
- `glb_sequence/`: frame별 GLB sequence
- `bundle_manifest.json`: 전체 산출물 경로와 summary
- `bundle_report.md`: 선택 사항, report 스크립트로 생성하는 Markdown 요약


2. 왜 필요한가
--------------

개별 exporter는 특정 포맷만 확인할 때 유용합니다. asset bundle exporter는 실제
작업 흐름에서 한 번의 simulation으로 viewer와 교환용 geometry 파일을 모두 만드는
용도입니다.

이 방식은 같은 parameter로 생성된 산출물이 한 manifest에 묶이므로, 나중에 결과를
비교하거나 재현하기 쉽습니다.


3. 실행 방법
------------

기본 실행:

    .\.venv\Scripts\python.exe export_spectral_choppy_asset_bundle.py

bundle report 생성:

    .\.venv\Scripts\python.exe report_spectral_choppy_asset_bundle.py --bundle-dir outputs\spectral_choppy_asset_bundle

작은 smoke test:

    .\.venv\Scripts\python.exe export_spectral_choppy_asset_bundle.py --size 64 --steps 48 --frame-every 12 --max-surface-points 32 --foam-threshold 0.0 --max-foam-points 64 --output-dir outputs\spectral_choppy_asset_bundle_smoke

고해상도 export:

    .\.venv\Scripts\python.exe export_spectral_choppy_asset_bundle.py --size 512 --steps 720 --frame-every 24 --max-surface-points 256 --output-dir outputs\spectral_choppy_asset_bundle_512


4. 주요 옵션
------------

- `--size`: 내부 FFT simulation grid 크기
- `--steps`: simulation step 수
- `--frame-every`: frame 저장 간격
- `--max-surface-points`: viewer/export surface의 한 축 최대 point 수
- `--choppiness`: 수평 displacement 강도
- `--foam-threshold`: foam point로 내보낼 최소 steepness
- `--max-foam-points`: frame당 최대 foam point 수
- `--output-dir`: bundle 저장 폴더


5. 한계
-------

이 bundle은 frame별 scene 파일 묶음입니다.

- glTF/GLB animation clip은 아직 만들지 않습니다.
- foam은 steepness 기반 point cloud입니다.
- 모든 포맷을 저장하므로 큰 grid와 많은 frame에서는 출력 크기가 빠르게 증가합니다.


6. 다음 개발 방향
-----------------

- animation clip이 포함된 glTF/GLB export
- Three.js/WebGPU viewer에서 bundle manifest를 직접 읽어 playback
- 더 물리적인 breaking wave와 spray layer 추가
