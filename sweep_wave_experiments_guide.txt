sweep_wave_experiments.py 가이드
================================

이 파일은 3D wave workflow를 실험 단위로 굴리기 위한 parameter sweep CLI입니다.

단일 dataset을 만드는 `export_wave_dataset.py`와 여러 dataset을 비교하는
`compare_wave_datasets.py`를 하나로 묶어, 여러 damping/gravity/CFL 조건을 한 번에
실행하고 비교 산출물까지 저장합니다.


1. 무엇을 만든 것인가
---------------------

`sweep_wave_experiments.py`는 다음 일을 순서대로 수행합니다.

- 지정한 parameter 값마다 bathymetry shallow-water simulation 실행
- 각 run을 `.npz` dataset으로 저장
- dataset metadata에 experiment name, run index, sweep parameter/value 기록
- baseline 대비 final-frame heatmap 생성
- frame별 L2/Linf CSV 생성
- frame별 L2/Linf Plotly chart 생성
- 전체 비교 Markdown table 생성
- 실험 전체 manifest JSON 저장

첫 번째 sweep 값이 baseline입니다. 이후 run은 이 baseline과 같은 grid/frame shape일 때
수치 차이를 계산합니다.


2. 실행 방법
------------

기본 damping sweep:

    .\.venv\Scripts\python.exe sweep_wave_experiments.py

작은 테스트용 sweep:

    .\.venv\Scripts\python.exe sweep_wave_experiments.py --experiment-name quick_damping_sweep --size 96 --steps 90 --frame-every 15 --values 0.9992,0.9994

gravity sweep:

    .\.venv\Scripts\python.exe sweep_wave_experiments.py --experiment-name gravity_sweep --parameter gravity --values 0.8,1.0,1.2

CFL sweep:

    .\.venv\Scripts\python.exe sweep_wave_experiments.py --experiment-name cfl_sweep --parameter cfl --values 0.25,0.35,0.45

velocity field까지 저장:

    .\.venv\Scripts\python.exe sweep_wave_experiments.py --store-velocity --experiment-name damping_velocity_sweep


3. 출력 파일
------------

기본 출력 폴더:

    outputs\sweeps\bathymetry_damping_sweep

그 안에는 다음 파일과 폴더가 생성됩니다.

- datasets\*.npz
- comparison.md
- manifest.json
- frame_metrics.html
- frame_metrics\*.csv
- diff_heatmaps\*.png

`manifest.json`은 실험 전체의 색인입니다. 어떤 parameter를 sweep했는지, 각 run의 dataset 경로가
무엇인지, 비교 산출물이 어디에 있는지를 기록합니다.


4. 왜 중요한가
--------------

3차원 파도 시뮬레이션은 보기 좋은 렌더링만으로는 충분하지 않습니다.

앞으로 grid size, damping, gravity, boundary, bathymetry, numerical scheme을 바꾸게 되면
각 변경이 수면장에 어떤 영향을 주는지 비교해야 합니다. 이 sweep CLI는 그 비교를
반복 가능한 실험 단위로 묶습니다.

이 구조의 장점:

- 실험 조건을 명시적으로 기록
- baseline 대비 차이를 자동 생성
- Plotly chart로 시간별 차이를 확인
- 대형 GPU run으로 가기 전에 작은 run으로 안정성 확인
- 나중에 3D viewer, benchmark, chunked dataset 저장으로 확장 가능


5. 현재 검증
------------

`validate_wave_workflow.py`에서 작은 2-run damping sweep을 실행해 다음을 확인합니다.

- 두 개의 validation dataset 생성
- comparison.md 생성
- frame_metrics.html 생성
- frame_metrics CSV 생성
- final-frame difference heatmap 생성
- baseline dataset 경로가 manifest에 기록되고 실제 파일이 존재


6. 현재 한계
------------

- 한 번에 하나의 parameter만 sweep합니다.
- multi-axis grid search는 아직 만들지 않았습니다.
- 큰 dataset은 여전히 NPZ 단일 파일로 저장합니다.
- sweep 결과를 한 화면에서 보는 dashboard는 아직 없습니다.


7. 다음 개발 방향
-----------------

다음 후보:

1. multi-parameter grid sweep
2. benchmark time/throughput을 manifest에 함께 기록
3. frame별 차이 peak 자동 탐지
4. sweep dashboard HTML 생성
5. 대형 run용 chunked dataset 저장


8. 한 줄 요약
--------------

`sweep_wave_experiments.py`는 여러 파도 시뮬레이션 조건을 자동 실행하고 비교 산출물을
모아 저장하는 실험 워크플로입니다.
